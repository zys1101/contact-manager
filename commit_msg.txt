fix: 修复头像上传功能故障 — multipart Content-Type、格式校验、URL显示

故障排查与修复:

1. api.ts upload() — Content-Type 覆盖问题 (根因)
   问题: 之前的修复使用 'Content-Type': undefined 来尝试让浏览器自动
   设置 multipart boundary，但 JavaScript 中 undefined 值在 axios
   config merge 时会被忽略，实例默认的 Content-Type: application/json
   可能保留，导致服务器无法解析 multipart 请求体
   修复: 在 transformRequest 中显式 delete headers['Content-Type']，
   确保浏览器完全自动生成 multipart/form-data 及 boundary 参数

2. Detail.tsx — 缺少 image/jpg MIME 类型检查
   问题: 前端仅检查 ['image/jpeg', 'image/png']，但部分系统将 .jpg
   文件报告为 image/jpg，导致合法文件被误拦截
   修复: 添加 image/jpg 到允许列表，与 Add.tsx 保持一致

3. Detail/Contacts/Blacklist — 头像 URL 无法显示
   问题: 后端返回 /uploads/avatar/xxx.jpg 相对路径，前端在 localhost:3000
   上渲染 <img src="/uploads/avatar/xxx.jpg" /> 时会请求 localhost:3000
   而非 localhost:8080/api，图片 404
   修复: 导出 api.ts 的 BASE_URL 常量，在三个页面中添加 getFullUrl()
   辅助函数，将相对路径拼接为基础URL

4. Detail.tsx — 文件 input 未重置
   问题: 上传成功后 file input 的值未清空，再次选择同一文件时不触发
   change 事件
   修复: 在 handleAvatarUpload finally 块中设置 e.target.value = ''

修改文件:
  - src/services/api.ts
  - src/pages/Contacts/Detail/Detail.tsx
  - src/pages/Contacts/Contacts.tsx
  - src/pages/Blacklist/Blacklist.tsx

验证: npm run build 编译成功，0 errors 0 warnings
